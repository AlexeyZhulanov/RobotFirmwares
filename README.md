# Universal Robot Project

## 📌 Общее описание проекта

Проект представляет собой универсальную модульную систему управления робототехническими устройствами с мобильного телефона.

Архитектура состоит из **трёх независимых прошивок**:

1. **UniversalRobotFirmware** — *основная (master) плата*
   Управляет устройствами, принимает команды от телефона и передаёт их slave-платам по RS-485.
2. **UnoSlaveFirmware** — *slave-плата (Arduino Uno)*
   Выполняет команды от master-платы и отправляет данные сенсоров.
3. **Esp32CamFirmware** — *камера*
   Независимо подключается к телефону и транслирует видеопоток.

📱 **Телефон** выступает как единый управляющий интерфейс:

* управляет устройствами через WebSocket,
* получает телеметрию,
* получает видеопоток напрямую от ESP32-CAM.

---

## 🧩 Архитектура системы

```
[ Телефон ]
   │
   ├── WebSocket (JSON)
   │
   ▼
[ ESP8266 / UniversalRobotFirmware ]
   │
   ├── GPIO (локальные устройства)
   ├── RS-485
   │
   ▼
[ Arduino Uno / UnoSlaveFirmware ]

[ ESP32-CAM ] ── WiFi ──► [ Телефон ]
```

---

# 🔥 UniversalRobotFirmware (MASTER)

**Основная и центральная прошивка проекта.**
Работает на ESP8266 и выполняет роль контроллера всей системы.

---

## 🔧 Основные функции

* Wi-Fi подключение к телефону
* Автообнаружение (UDP discovery)
* WebSocket-сервер для управления и телеметрии
* Управление локальными GPIO-устройствами
* Управление удалёнными устройствами по RS-485
* Хранение конфигурации в SPIFFS (`/config.json`)
* Унифицированный JSON-протокол
* Поддержка:

  * LED
  * моторов (PWM + направление)
  * цифровых и аналоговых сенсоров
  * устройств на slave-платах (`uno_*`)

---

## 🌐 Сеть и протоколы

| Назначение       | Протокол  | Порт      |
| ---------------- | --------- | --------- |
| HTTP (статус)    | HTTP      | 80        |
| Управление       | WebSocket | 81        |
| Автообнаружение  | UDP       | 4210      |
| Slave-устройства | RS-485    | 9600 baud |

## 📡 UDP Discovery

Master-плата каждые 3 секунды отправляет:

```
I_AM_ROBOT:<IP_АДРЕС>
```

📌 Отправка прекращается, когда телефон подключается по WebSocket.

---

## 🔌 WebSocket API (основной интерфейс)

Все команды и ответы — **JSON**.

---

### 📥 Команды от телефона

#### `get_devices`

Запрос списка устройств.

```json
{ "cmd": "get_devices" }
```

📤 Ответ:

```json
{
  "devices": [
    { "name": "led1", "pin": 5, "pin2": -1, "type": "led", "state": false }
  ]
}
```

---

#### `get_status`

Статус платы.

```json
{ "cmd": "get_status" }
```

📤 Ответ:

```json
{
  "cmd": "status",
  "uptime_ms": 123456,
  "device_count": 3
}
```

---

#### `get_board_info`

Информация о плате.

📤 Ответ:

```json
{
  "cmd": "board_info",
  "board": "ESP8266",
  "chip_id": 12345678,
  "cpu_freq_mhz": 80
}
```

---

#### `set_config`

Загрузка конфигурации устройств (и сохранение в SPIFFS).

```json
{
  "cmd": "set_config",
  "config": {
    "devices": [
      { "name": "led1", "type": "led", "pin": 5 },
      { "name": "motor1", "type": "motor", "pin": 4, "pin2": 14 }
    ]
  }
}
```

---

#### `action`

Управление устройством.

**Включение / выключение**

```json
{
  "cmd": "action",
  "device": "led1",
  "action": "on"
}
```

**Управление мотором**

```json
{
  "cmd": "action",
  "device": "motor1",
  "action": "set_speed",
  "value": -120
}
```

📌 Диапазон скорости: `-255 … 255`

---

#### `get_sensors`

Снимок текущих значений сенсоров.

```json
{ "cmd": "get_sensors" }
```

---

#### `subscribe`

Подписка на сенсор.

```json
{
  "cmd": "subscribe",
  "device": "sensor1"
}
```

---

## 📤 Асинхронные события (от master к телефону)

#### `device_state`

```json
{ "cmd": "device_state", "name": "led1", "state": 1 }
```

#### `sensor_update`

```json
{ "cmd": "sensor_update", "device": "sensor1", "value": 0 }
```

#### `speed_changed`

```json
{ "cmd": "speed_changed", "device": "motor1", "value": 120 }
```

#### `uno_sensors`

Данные сенсоров со slave-платы.

```json
{
  "cmd": "uno_sensors",
  "id": 1,
  "data": "512,1023,44"
}
```

## 🔗 RS-485 (Master → Slave)

### Формат пакетов

```
<ID>:<COMMAND>:<VALUE>\n
```

### Примеры

```
1:uno_motor_9_8:-255
1:uno_led13:1
1:GET_SENSORS:0
```

📌 Команды автоматически троттлятся (не чаще 5 раз/сек).

# 🧱 UnoSlaveFirmware (SLAVE)

Прошивка для Arduino Uno, управляемая по RS-485.

## Возможности

* Управление моторами без драйвера
* Управление светодиодами
* Считывание аналоговых сенсоров A0–A5
* Ответы master-плате

---

## Поддерживаемые команды

### `GET_SENSORS`

📤 Ответ:

```
1:SENSORS:512,300,1023,0,0,900
```

---

### `uno_motor_<pwmPin>_<dirPin>`

Пример:

```
1:uno_motor_9_8:-200
```

---

### `uno_led<pin>`

Пример:

```
1:uno_led13:1
```

---

# 📷 Esp32CamFirmware (CAMERA)

Независимая прошивка для ESP32-CAM.

## Возможности

* MJPEG-видеопоток
* Автообнаружение по UDP
* Прямое подключение телефона

## Поток

```
http://<IP_КАМЕРЫ>:81
```

📌 Формат: `multipart/x-mixed-replace (MJPEG)`
📌 Частота: ~30 FPS

---

## 📡 UDP Discovery (камера)

```
I_AM_CAMERA:<IP_АДРЕС>
```
